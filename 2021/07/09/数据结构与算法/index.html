一、二分查找

1、特点

二分查找也称折半查找（Binary Search），是一种在有序数组中查找某一特定元素的搜索算法。我们可以从定义可知，运用二分搜索的前提是数组必须是有序的，这里需要注意的是，我们的输入不一定是数组，也可以是数组中某一区间的起始位置和终止位置。

此外，取整方向很重要，

  • 区间划分为[left,…,mid 丨mid + 1,…,right]，中点mid属于左半区间时，mid向下取整，mid = floor(l + (r - l) / 2)；即mid = l + (r - l) / 2
  • 区间划分为[left,…,mid - 1 丨 mid,…,right]，中点mid属于右半区间时，mid向上取整，mid = ceil(l + (r - l) / 2)；即mid = l + (r - l + 1) / 2

还有一种思路判断是否死循环，即l与r相差为1时，代入mid计算公式。

  1. 如果mid = l + (r - l) / 2 = l ，则下面的条件里 l = mid + 1，否则会mid = l, l = mid无限循环.相应的，r = mid，区间划分为[left,…,mid 丨mid + 1,…,right]
  2. 如果mid = l + (r - l + 1) / 2 = r , 则下面的条件里 r = mid - 1, 否则会mid = r, r = mid无限循环.相应的，l = mid，区间划分为[left,…,mid - 1 丨 mid,…,right]

若取错会导致l, r更新不了，就会导致死循环

写法总结

while (left <= right) 与 while (left < right) 写法的区别

首先

  • while (left <= right) 在推出循环的时候有 left == right + 1，思路类似于在循环体内部查找元素
  • while (left < right) *在退出循环的时候有left == right*，思路类似于在循环体内部排除元素**

简单的二分问题，用第一个，如果题目要我们找的数的性质很简单，可以用这种写法，在循环体里找到了就退出

在一些复杂问题中，例如找一些边界的值，用 while(left < right) 其实是更简单的，可以归纳为「左右边界向中间走，两边夹」，这种思路在解决复杂问题的时候，可以使得思考的过程变得简单。把要找的数留到最后，在退出循环以后做判断。最重要的原因是退出循环以后有 left == right 成立，这种思考问题的方式不容易出错。

while(left < right) 写法难点：初学时不注意就会出现死循环。需注意分支和中间数的取法。建议使用这种的方式去解决一些较困难的问题。

此外，若题目要求满足条件的最小或者满足条件的最大，这种题一般是二分

step1：状态定义

f(node)(1)表示选择node节点的情况下，i节点子树的最大权值和，用l,r表示i的左右孩子，此时选择node节点，就不能选择左右孩子。

step2：状态转移方程

选择当前节点，f(node)(1) = f(l)(0) + f(r)(0) + node.val

不选择当前节点，f(node)(0) = Math.max(f(l)(0), f(l)(1)) + Math.max(f(r)(0), f(r)(1))

一个结点都没有，空节点，返回 0，对应后序遍历时候的递归终止条件；

step4：输出

在根结点的时候，返回两个状态的较大者。

3、前缀树（字典树）

leetcode208.实现Trie(前缀树)

Trie，又称前缀树或字典树，是一棵有根树，其每个节点包含以下字段：

  • 指向子节点的指针数组 children。对于本题而言，数组长度为 26，即小写英文字母的数量。此时 children[0] 对应小写字母 a，children[1] 对应小写字母 b，…，children[25] 对应小写字母 z。
  • 布尔字段 isEnd，表示该节点是否为字符串的结尾。

插入字符串

我们从字典树的根开始，插入字符串。对于当前字符对应的子节点，有两种情况：

  • 子节点存在。沿着指针移动到子节点，继续处理下一个字符。
  • 子节点不存在。创建一个新的子节点，记录在children 数组的对应位置上，然后沿着指针移动到子节点，继续搜索下一个字符。

重复以上步骤，直到处理字符串的最后一个字符，然后将当前节点标记为字符串的结尾。

查找前缀

我们从字典树的根开始，查找前缀。对于当前字符对应的子节点，有两种情况：

  • 子节点存在。沿着指针移动到子节点，继续搜索下一个字符。
  • 子节点不存在。说明字典树中不包含该前缀，返回空指针。

重复以上步骤，直到返回空指针或搜索完前缀的最后一个字符。

若搜索到了前缀的末尾，就说明字典树中存在该前缀。此外，若前缀末尾对应节点的 isEnd 为真，则说明字典树中存在该前缀字符串。

双向bfs与单向bfs的对比

在朴素的 BFS 实现中，空间的瓶颈主要取决于搜索空间中的最大宽度。如果宽度太大，会带来搜索空间爆炸问题。

比如对于127.单词接龙，对于一个长度为 10 的 beginWord 替换一次字符可以产生 10 * 25 个新单词（每个替换点可以替换另外 25 个小写字母），第一层就会产生 250 个单词；第二层会产生超过 6 * 104  个新单词 …

对于「有解」、「有一定数据范围」同时「层级节点数量以倍数或者指数级别增长」的情况，「双向 BFS」的搜索空间通常只有「朴素 BFS」的空间消耗的几百分之一，甚至几千分之一。
